# Android XR Design Guidelines

Fetched from: https://developer.android.com/design/ui/xr
Date: 2026-05-21

## Core Principles

1. Start from where you are — multitask in real world or escape into full immersion
2. Keep users comfortable — prioritize comfort in extended use
3. Add key XR moments — leverage spatial capabilities
4. Make your app accessible — ensure accessibility across all XR

## Spatial UI Design

- Break apps into spatial panels
- Include orbiters for navigation
- Add spatial elevation for depth perception

## 3D Content

- Add 3D elements for depth, scale, realism
- Design tangible content that responds to interaction

## Environments

- Put user at center with full immersion
- Design dynamic environments adapting to user presence
- Create spatial audio to complement visuals

## Visual Design

- Adopt Material Design 3 styles
- Large target sizes for comfortable interaction
- XR-appropriate colors and typography

## Multimodal Inputs

- Natural interaction methods
- Multiple input modalities
- Gesture-based interactions

## Material Design for XR

- Components and layouts adapt for XR
- Large-screen apps following Material guidelines are compatible
- Figma design kit available
