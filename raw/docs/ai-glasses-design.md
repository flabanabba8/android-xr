# AI Glasses Design Guidelines

Fetched from: https://developer.android.com/design/ui/ai-glasses
Date: 2026-05-21

## Core Principles

- All-day wear: comfortable extended use
- Hands-free access: interaction without manual input
- Focus on presence: enhance real-world context, don't obstruct
- Glanceable information: data consumed at a glance

## Key Design Areas

1. World Sensing — respond to real-world context via sensors
2. Faster Comprehension — visuals in context, minimize cognitive load
3. Visual Assistance — augmented capabilities (superpowers)
4. Hands-Free Capture — stay in the moment, passive data collection

## Audio AI Glasses

- Primary interaction: natural language voice
- Output: text-to-speech, audio guidance
- Use cases: media control, navigation, task assistance
- Design focus: concise audio responses

## Display AI Glasses

- Primary interaction: voice + visual + gesture
- Output: high-contrast minimal visuals
- Subtle, glanceable, appears only when relevant
- Combine audio with contextual visuals

## Jetpack Compose Glimmer Design Language

- High contrast for readability on small see-through displays
- Minimal, clean aesthetic
- Optimized for glanceable content
- Modern component library

## Philosophy

AI Glasses apps should: enrich daily life hands-free, provide discreet digital access, enhance focus and presence, minimize visual obstruction, support conversational interaction, respond intelligently to context.

Key differentiator: ambient intelligence + contextual awareness while maintaining physical world presence.
